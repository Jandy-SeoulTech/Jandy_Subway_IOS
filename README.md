# Jandy Subway IOS

### commit message convention
```text
- feat: 새로운 기능 추가
- fix: 버그 수정
- refactor: 코드 리팩토링
- style: 코드 스타일 변경(ex 들여쓰기 바꾸기, 세미콜론 넣기)
- chore: 설정 변경
```
### branch naming convention
```text
- main 
- develop
- feature/XXX ex) feature/login
- release-XXX ex) release-1.0.0
- hotfix-XXX ex) hotfix-login
```
